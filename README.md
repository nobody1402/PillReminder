# یادآور قرص (Pill Reminder)

اپلیکیشن اندروید (Kotlin + Jetpack Compose + Room + AlarmManager) برای مدیریت دقیق مصرف چند دارو.

## چطور بسازمش؟

1. [Android Studio](https://developer.android.com/studio) (نسخه Koala یا جدیدتر) را نصب کنید.
2. پوشه `PillReminder` را با گزینه **Open** در Android Studio باز کنید (نه Import).
3. صبر کنید Gradle Sync تمام شود (نیاز به اینترنت برای دانلود کتابخانه‌ها دارد).
4. یک دستگاه/شبیه‌ساز با Android 8 (API 26) یا بالاتر انتخاب کنید و Run بزنید.

## چه چیزی پیاده‌سازی شده

- **ثبت قرص:** نام، عکس (مسیر فایل ذخیره می‌شود)، رنگ، مقدار مصرف (کامل/نصف/یک‌چهارم/چند عدد)، رابطه با غذا، ساعات مصرف، تعداد روزهای درمان.
- **آلارم دقیق:** با `AlarmManager.setExactAndAllowWhileIdle` — حتی در Doze Mode فعال می‌شود، و بعد از ری‌استارت گوشی با `BootReceiver` دوباره برنامه‌ریزی می‌شود.
- **نوتیفیکیشن غنی:** ساعت، نام دارو، مقدار دقیق مصرف، وضعیت غذا و زمان انتظار — با سه دکمه: ✅ مصرف کردم / ⏰ ۱۰ دقیقه بعد / ❌ رد شد.
- **صفحه «امروز»:** خلاصه وضعیت مصرف هر دوز با آیکون ✅ / ❌ / 🕗.
- **قوانین تداخل زمانی:** تعریف حداقل فاصله بین دو دارو (`InteractionChecker`)؛ هنگام ذخیره یک دارو با ساعت‌های ناسازگار، هشدار متنی نمایش داده می‌شود.
- **موجودی دارو:** با هر بار زدن «مصرف کردم» موجودی کم می‌شود؛ اگر به آستانه هشدار برسد، نوتیفیکیشن جداگانه می‌آید. یک `WorkManager` هم هر ۲۴ ساعت به‌صورت پیش‌گیرانه موجودی همه داروها را چک می‌کند.
- **حالت سالمندان:** در تنظیمات قابل فعال‌سازی — فونت بزرگ‌تر در صفحه امروز، ویبره قوی‌تر، نوتیفیکیشن ماندگارتر (`setOngoing`)، و خواندن نام دارو با TTS فارسی (در صورت پشتیبانی گوشی).

## چیزهایی که باید تکمیل/تست کنید

این یک اسکلت کاملاً کارکردی از معماری اصلی است، اما چون در محیط من نه SDK اندروید نصب است و نه دسترسی اینترنت برای Gradle، **کد کامپایل یا روی گوشی واقعی تست نشده است**. قبل از استفاده واقعی حتماً:

- در Android Studio یک‌بار Build/Run کنید و خطاهای احتمالی کامپایل (که معمولاً جزئی هستند: import گم‌شده، نسخه کتابخانه) را برطرف کنید.
- گرفتن عکس واقعی از دوربین (فعلاً فقط فیلد `photoUri` در مدل هست؛ باید `ActivityResultContracts.TakePicture` را به فرم افزودن دارو وصل کنید).
- انتخاب‌گر ساعت گرافیکی (`TimePicker`) به‌جای فیلد متنی برای ساعات مصرف.
- روی گوشی‌های شیائومی/هواوی و... باید Battery Optimization برای اپ خاموش شود تا AlarmManager قابل‌اعتماد بماند (این محدودیت سیستم‌عامل است، نه باگ اپ).
- درخواست مجوز `SCHEDULE_EXACT_ALARM` در Android 12+ باید به کاربر با یک دیالوگ توضیح داده شود (کد فعلی فرض می‌کند مجوز از قبل داده شده و در غیر این صورت به آلارم غیردقیق سقوط می‌کند).

## ساختار پوشه‌ها

```
app/src/main/java/com/example/pillreminder/
  data/      -> Entity ها، DAO ها، Room Database، Repository
  alarm/     -> AlarmScheduler، AlarmReceiver، نوتیفیکیشن، بررسی موجودی، BootReceiver
  util/      -> فرمت متن فارسی، بررسی تداخل دارویی، تنظیمات، TTS، ساخت برنامه امروز
  ui/        -> MainActivity و تمام صفحات Compose
```
